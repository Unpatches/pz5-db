# pz5-db

Задачи проекта
1.	Установить и настроить PostgreSQL локально.
2.	Подключиться к БД из Go с помощью database/sql и драйвера PostgreSQL.
3.	Выполнить параметризованные запросы INSERT и SELECT.
4.	Корректно работать с context, пулом соединений и обработкой ошибок.

---

## Установка и запуск

(Необходимы предустановленные Go, Git и PostgreSQL)

Клонировать репозиторий:

```
git clone https://github.com/Unpatches/pz5-db
cd pz5-db
```

Команда запуска сервера:

```
go run .
```

## Краткие ответы
1) sql.Open возвращает объект *sql.DB (это пул соединений, а
не одно соединение) и необходим для подключения к БД. Важно что sql.Open() не устанавливает соединение немедленно, а только
настраивает пул.

2) $1, $2 — параметры запроса (placeholders). Это защищает от SQL-инъекций

3) Для чтения одной строки используют метод QueryRow().
   Для чтения нескольких строк используют метод Query().
   Для команд, которые не возвращают результат, используется метод Exec().

4) Транзакция — это последовательность операций, которая выполняется как единое целое.
   Пример: перевод денег между счетами.
   Свойства транзакций описываются аббревиатурой ACID:
   - Atomicity (Атомарность) — все операции выполняются либо полностью, либо не выполняются вовсе.
   - Consistency (Согласованность) — данные всегда остаются в корректном состоянии.
   - Isolation (Изолированность) — параллельные транзакции не мешают друг другу.
   - Durability (Надёжность) — после фиксации данные сохраняются даже при сбое.
     
   - db.SetMaxOpenConns(10) - максимум 10 соединений
   - db.SetMaxIdleConns(5) - до 5 неактивных соединений
   - db.SetConnMaxLifetime(30 * time.Minute) - срок жизни соединения
   10 соединений достаточно для учебной/локальной нагрузки.

   Idle-соединения ускоряют повторные запросы.

   Lifetime обновляет соединения, чтобы избежать зависаний.
